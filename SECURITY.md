# Security Policy

## Supported Versions

Only the current `main` branch is actively maintained.

## Reporting Security Issues

Do not open a public issue for vulnerabilities that expose private data, enable cross-site scripting, or bypass access controls.

Report privately to the repository owner through GitHub security advisories if enabled, or by direct contact if the repository owner provides one.

## Scope

Security fixes are welcome for:

- Extension popup DOM injection.
- Unsafe URL handling.
- Overly broad data persistence.
- Accidental leakage of request headers or logs.
- Browser-extension permission reduction.

Requests to bypass DRM, paywalls, private content, or authentication gates are out of scope.
