# Contributing

Thanks for considering a contribution.

## Guidelines

- Keep changes scoped and testable.
- Do not add code intended to bypass DRM, paywalls, private access controls, or platform security.
- Prefer detection improvements that are generic or clearly documented by URL/content-type behavior.
- Add or update tests for shared detection helpers when adding new URL patterns.
- Run the syntax checks and tests listed in `README.md` before opening a pull request.

## Reporting Site Breakage

When reporting a site that fails, include:

- Browser and version.
- Extension version.
- Site URL or a minimal public sample URL.
- Whether the resource appears as direct download, HLS, DASH, or recording-only.
- Logs copied from the extension popup when possible.

Do not include private media URLs, credentials, cookies, or paid/private content.
